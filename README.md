# zmk-config-keyball39

ZMK config for the [Keyball 39](https://github.com/Yowkees/keyball) — ported from a QMK ferris/sweep Colemak-DH config (`mustardthepirate`).

- **Alpha layout:** Colemak-DH
- **Host OS layout:** UK ISO (Linux / omarchy)
- **Home-row mods:** Shift on `A` / `O`
- **Home-row layer-taps:** `R`/`S`/`T`/`N`/`E`/`I`
- **Trackball** handles pointer movement; mouse layer covers clicks + scroll

`<HRM>` = home-row mod (shift on tap-and-hold). `<N>` = layer tap (tap = letter, hold = layer N). `C()`/`A()`/`G()` = Ctrl/Alt/Gui + key.

## Legend

| Symbol | Meaning |
|--------|---------|
| `▽`    | transparent (falls through to lower active layer) |
| `∅`    | no-op |
| `sk X` | sticky / one-shot modifier (applies to next key only) |
| `<N>`  | hold = layer N, tap = letter |
| `C()`  | LCtrl + key · `A()` LAlt · `G()` LGui · `AG()` LAlt+LGui · `SG()` Shift+Gui |
| `A-T(,)` | mod-tap: tap = `,`, hold = RAlt |

## Building

On every push, GitHub Actions produces three UF2s: `keyball39_left`, `keyball39_right`, `settings_reset`. Download from the workflow run artifacts.

## Flashing

1. Double-tap reset → half mounts as `NICENANO`.
2. First time (or after keymap breaks BT): flash `settings_reset.uf2` to both halves, then the real firmware.
3. The central half is the one that pairs to the host; BT profile keys (FN layer) only fire on the central side.
